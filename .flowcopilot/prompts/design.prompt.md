## Step 1 · 全栈代码扫描

### 后端层级（由外至内依次扫描）

| 层级       | 扫描目标路径                           | 关注内容                                   |
| ---------- | -------------------------------------- | ------------------------------------------ |
| Controller | `${context.backendPaths.controller}` | 路由注册、请求映射、参数校验、接口文档注释 |
| Service    | `${context.backendPaths.service}`    | 业务逻辑、事务管理、服务间调用关系         |
| DAO/Mapper | `${context.backendPaths.dao}`        | SQL 映射、查询方法、分页逻辑、关联查询     |
| Entity     | `${context.backendPaths.entity}`     | 实体字段定义、字段注解、关联关系映射       |

### 前端层级（由顶至底依次扫描）

| 层级       | 扫描目标路径                            | 关注内容                                   |
| ---------- | --------------------------------------- | ------------------------------------------ |
| Views      | `${context.frontendPaths.views}`      | 页面路由、布局结构、页面生命周期           |
| Components | `${context.frontendPaths.components}` | 组件树结构、Props 定义、事件通信、复用模式 |
| Store      | `${context.frontendPaths.store}`      | 状态模块划分、actions/getters 定义、持久化 |
| API        | `${context.frontendPaths.api}`        | 接口封装、请求拦截、类型定义、错误处理     |

> 扫描要求：不仅阅读代码，还需理解"当前项目的架构风格、命名约定和设计范式"，后续所有新增代码须与之保持一致。

## Step 2 · 数据底座分析

使用可用的数据库工具（如 MCP MySQL 工具）完整分析数据层：

1. **表结构梳理**：列出所有相关数据表、字段名称、字段类型、约束条件
2. **关联关系图谱**：主键/外键关联、逻辑关联（如通过 code/type 字段的隐式关联）
3. **数据字典**：枚举值含义、状态机流转规则、数据生命周期
4. **索引与性能**：现有索引设计、慢查询风险评估

---

## Step 3 · 差异分析（Gap Analysis）

将目标功能需求与现有项目基线对照，按以下维度输出分析矩阵：

| 类别                   | 说明                                      |
| ---------------------- | ----------------------------------------- |
| 🟢**可直接复用** | 现有模块/组件无需修改即可满足需求的部分   |
| 🟡**需修改扩展** | 现有模块基本满足但需要扩展字段/逻辑的部分 |
| 🔴**需全新开发** | 现有系统中不存在、需要从零构建的部分      |

对每项差异，标注影响范围（涉及哪些文件/表/接口）和预估工作量。

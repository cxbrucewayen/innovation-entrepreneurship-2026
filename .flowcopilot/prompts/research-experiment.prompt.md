## Step 7 · 实验运行规范

### 7a. 可复现性保障

```yaml
random_seed: ${context.experimentConfig.seed}
gpu_device: ${context.experimentConfig.gpu}
pytorch_deterministic: true
cudnn_benchmark: false
num_workers: 4
```

### 7b. 统计显著性检验

- 每组实验至少运行 3-5 次（不同随机种子）
- 报告均值 ± 标准差
- 使用配对 t-检验或 Wilcoxon 符号秩检验验证显著性
- 显著性水平：p < 0.05

### 7c. 实验结果目录规范

**目录层次结构**：

```
experiments/
├── results/
│   ├── {order:02d}_{category}_{name}_{dataset}_{YYYYMMDD_HHmmss}/   ← 每个实验独立文件夹
│   │   ├── config.yaml          # 完整实验配置快照（命令行参数 + yaml）
│   │   ├── metrics.json         # 结构化评估指标结果
│   │   ├── logs/                # 训练与评估日志（train.log, eval.log）
│   │   ├── figures/             # 本实验图表（可视化结果）
│   │   └── summary.md           # 本实验执行摘要
├── figures/                     # 全局图表（跨实验对比图，供论文使用）
└── reports/                     # 最终实验报告
```

**文件夹命名规则**：`{执行序号:02d}_{实验类型}_{方法或配置名}_{数据集}_{YYYYMMDD_HHmmss}`

| 字段 | 格式 | 可取值 | 示例 |
|------|------|-------|------|
| 执行序号 | 两位补零整数，对应 Phase 6 步骤编号 | - | `01`, `03`, `12` |
| 实验类型 | 小写英文 | `baseline` / `ours` / `ablation` / `hyperparam` / `efficiency` / `qualitative` / `summary` | `ablation` |
| 方法或配置名 | 小写下划线，简短清晰 | - | `resnet50`, `full_model`, `wo_attention`, `lr_1e3` |
| 数据集 | 小写下划线，多数据集时用 `all` | - | `cifar10`, `imagenet`, `all` |
| 时间戳 | `YYYYMMDD_HHmmss`（执行时生成，用于区分多次重跑） | - | `20260326_143022` |

**命名示例**：

```
01_baseline_resnet50_cifar10_20260326_143022/
02_baseline_vit_b16_cifar10_20260326_151134/
03_ours_full_model_cifar10_20260326_162215/
04_ablation_wo_module_a_cifar10_20260326_173401/
05_ablation_wo_module_b_cifar10_20260326_181523/
06_hyperparam_lr_sweep_cifar10_20260326_183022/
07_efficiency_latency_compare_all_20260326_190512/
08_qualitative_case_study_cifar10_20260326_200103/
09_summary_main_results_all_20260326_203011/
```

> **核心原则**：每次启动实验前先生成时间戳，所有输出（日志、权重、结果、图表）均保存至对应独立文件夹，**严禁**不同实验混存于同一目录。

### ⚠️ 分阶段写入策略（防超时 — 强制执行）

由于输出内容较长，一次性写入会导致连接超时断开。**必须**严格按以下流程分阶段多次调用工具写入：

**写入目标**：先写入临时文件 `temp.md`，全部完成后重命名为最终文件。

1. **第 1 次调用** `create_file` → 创建 `temp.md`，写入前 200 行内容（标题 + 实验目标 + 数据集 + 评估指标）
2. **确认文件存在** → 使用 `read` 工具确认 `temp.md` 已生成
3. **第 2 次调用** `edit` → 以 `temp.md` 末尾 3-5 行为 oldString 锚点，替换为「这几行 + 新增 200 行」（基线方法、实验计划）
4. **第 3 次调用** `edit` → 继续追加 200 行（实验环境、训练策略）
5. **第 N 次调用** `edit` → 继续追加 200 行（结果展示模板、实验时间规划），直到内容全部写完
6. **最终验证** → 使用 `read` 工具检查 `temp.md` 完整性
7. **重命名** → 使用 `execute` 工具执行重命名命令，将 `temp.md` 重命名为 `${output.path}`

> ⛔ 每次工具调用写入内容 **严禁超过 200 行**，否则会超时断连！
> ⛔ 必须多次调用工具，先确认前一段写入成功再继续下一段！

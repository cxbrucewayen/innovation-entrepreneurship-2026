## Step 2 · 论文检索与筛选

### 2a. 使用 MCP 工具搜索论文

依次通过以下方式搜索相关论文：

1. **GitHub MCP 搜索代码仓库**：
   - 使用 `search` 工具搜索与课题相关的开源项目和代码实现
   - 关注 README 中引用的论文和方法

2. **Web Fetch 搜索学术数据库**：
   - 使用 `web/fetch` 工具访问以下学术搜索引擎：
     - arXiv API: `https://export.arxiv.org/api/query?search_query=all:{关键词}&max_results=20`
     - Semantic Scholar API: `https://api.semanticscholar.org/graph/v1/paper/search?query={关键词}&limit=20&fields=title,abstract,year,authors,citationCount,url,externalIds`
     - DBLP API: `https://dblp.org/search/publ/api?q={关键词}&format=json&h=20`
   - 对每个数据源至少执行 2-3 组不同关键词组合的搜索

3. **滚雪球法扩展**：从高被引论文的参考文献中发现更多相关工作

### 2b. 论文筛选标准

| 优先级 | 筛选条件 |
|--------|---------|
| 🔴 必选 | 与核心研究问题直接相关的方法/模型论文 |
| 🟡 重要 | 高被引综述论文（了解领域全貌） |
| 🟢 参考 | 相关数据集、评估基准、应用案例 |

目标：筛选出 ${context.maxPapers} 篇高质量相关论文。

## Step 3 · 论文精读与分析

### 3a. 使用 Playwright 阅读论文

对筛选出的每篇重要论文：

1. 使用 `web/fetch` 获取论文页面内容（arXiv abstract page, PDF链接等）
2. 如有 HTML 版本，优先阅读 HTML 全文
3. 提取以下信息：

| 分析维度 | 提取内容 |
|---------|---------|
| 基本信息 | 标题、作者、期刊/会议、年份、DOI |
| 研究问题 | 论文要解决什么问题？动机是什么？ |
| 方法核心 | 提出了什么方法/模型/算法？核心创新点 |
| 技术细节 | 网络架构、损失函数、训练策略等关键技术 |
| 实验设置 | 数据集、评估指标、基线方法、实验环境 |
| 主要结果 | 关键性能数据、对比结果、消融实验结论 |
| 局限性 | 作者自述的局限或你识别出的不足 |
| 代码可用性 | 是否开源？GitHub 链接？ |

### 3b. 论文间关系分析

- **方法演进脉络**：按时间线梳理方法的发展演进
- **技术路线对比**：不同方法的异同点、优劣势
- **引用关系**：哪些论文是奠基性工作？哪些是最新进展？

## Step 4 · 研究空白识别

基于文献分析，识别以下研究机会：

| 类型 | 说明 |
|------|------|
| 🔬 **未解决问题** | 现有方法仍未攻克或效果不佳的问题 |
| 📈 **可改进空间** | 现有方法的明确瓶颈或可优化点 |
| 🔗 **交叉融合机会** | 不同技术路线的潜在结合点 |
| 📊 **缺失的实验** | 领域内缺乏的对比实验或验证场景 |

## 输出要求

### ⚠️ 分阶段写入策略（防超时 — 强制执行）

由于输出内容较长，一次性写入会导致连接超时断开。**必须**严格按以下流程分阶段多次调用工具写入：

**写入目标**：先写入临时文件 `temp.md`，全部完成后重命名为最终文件。

1. **第 1 次调用** `create_file` → 创建 `temp.md`，写入前 200 行内容（标题 + 研究背景 + 检索策略 + 文献概览）
2. **确认文件存在** → 使用 `read` 工具确认 `temp.md` 已生成
3. **第 2 次调用** `edit` → 以 `temp.md` 末尾 3-5 行为 oldString 锚点，替换为「这几行 + 新增 200 行」（每篇论文精读）
4. **第 3 次调用** `edit` → 继续追加 200 行（技术对比表、方法演进脉络）
5. **第 N 次调用** `edit` → 继续追加 200 行（研究空白、关键数据集等），直到内容全部写完
6. **最终验证** → 使用 `read` 工具检查 `temp.md` 完整性
7. **重命名** → 使用 `execute` 工具执行重命名命令，将 `temp.md` 重命名为 `${output.path}`

> ⛔ 每次工具调用写入内容 **严禁超过 200 行**，否则会超时断连！
> ⛔ 必须多次调用工具，先确认前一段写入成功再继续下一段！

同样，BibTeX 文件写入也使用 `temp.bib` → 重命名为 `${context.outputDir}/references.bib` 的方式。

### 输出文件 2: BibTeX 参考文献 — 保存到 `${context.outputDir}/references.bib`

使用 `edit` 工具创建 BibTeX 文件，包含所有引用论文的完整引用信息：

```bibtex
@article{AuthorYear_keyword,
  title     = {论文标题},
  author    = {作者列表},
  journal   = {期刊名 / booktitle = {会议名}},
  year      = {年份},
  volume    = {卷号},
  pages     = {页码},
  doi       = {DOI},
  url       = {URL}
}
```

⚠️ **重要提示**：
1. **必须**使用 `edit` 或 `create_file` 工具写入文件，**禁止**使用终端命令行重定向。
2. BibTeX 条目的 key 格式统一为 `第一作者姓Year_关键词`（如 `Wang2024_transformer`）。
3. 所有文件使用 **UTF-8** 编码。

> 注意：本阶段输出的文献综述是后续 Phase 2（创新方案设计）的核心输入。务必全面、准确、有洞察力。

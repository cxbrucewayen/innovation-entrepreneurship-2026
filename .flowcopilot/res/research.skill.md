# 自动化科研工作流技能 (Research Automation Skill)

将用户的研究课题描述自动化为完整的科研项目，覆盖从文献调研到实验分析的全生命周期。

## 适用场景

- 用户提出一个研究课题/方向，需要系统性地开展研究
- 需要全面的文献检索和综述
- 需要设计创新方案并撰写方法论
- 需要搭建实验框架和执行实验
- 需要生成论文级别的实验报告和图表

## 不适用场景

- 单纯的代码 Debug（使用默认 Agent）
- 与科研无关的软件开发（使用 dev-workflow）
- 只需要查找一篇论文（直接使用 web/fetch）

## 参数说明

- `$0`: 研究课题描述（必需），应包含：
  - 研究方向/领域
  - 核心问题/目标
  - 关键技术约束（如框架偏好、数据限制等）
- `--lang <语言>`: 代码实现语言（默认 Python）
- `--framework <框架>`: 深度学习框架（默认 PyTorch）
- `--max-papers <数量>`: 最大检索论文数（默认 20）

## 执行流程详细说明

### Phase 1: 文献检索与综述

**目标**：建立研究课题的完整文献认知

**MCP 工具使用策略**：

1. **arXiv API 搜索**：
   ```
   web/fetch: https://export.arxiv.org/api/query?search_query=all:{keywords}&max_results=20
   ```
   - 提取标题、摘要、作者、链接
   - 关注最近 2-3 年的新进展

2. **Semantic Scholar API 搜索**：
   ```
   web/fetch: https://api.semanticscholar.org/graph/v1/paper/search?query={keywords}&limit=20&fields=title,abstract,year,authors,citationCount,url,externalIds
   ```
   - 按引用量排序，找高影响力论文
   - 利用 citation graph 进行滚雪球

3. **DBLP API 搜索**：
   ```
   web/fetch: https://dblp.org/search/publ/api?q={keywords}&format=json&h=20
   ```
   - 确认论文的会议/期刊来源

4. **Playwright 阅读论文**：
   - 对高优先级论文，使用 Playwright 访问 arXiv HTML 版本或论文官方页面
   - 提取方法细节、实验设置、核心结果

**交付物**：
- `references.md` — 包含每篇论文的结构化分析 + 技术路线对比 + 研究空白
- `references.bib` — 标准 BibTeX 格式参考文献

---

### Phase 2: 创新方案设计

**目标**：基于研究空白提出原创性技术方案

**关键执行步骤**：

1. 从文献综述中提取所有研究空白
2. 对每个空白进行可行性评估
3. 提出 2-3 个候选创新方案
4. 使用 MCP 搜索相关开源框架评估可行性
5. 选择最优方案，撰写完整方法论

**框架搜索策略**：
- GitHub: 搜索相关方法的开源实现
- Papers with Code: 查看 SOTA 方法的代码可用性
- Hugging Face: 搜索可复用的预训练模型

**交付物**：
- `methodology.md` — 完整方法论文档（含架构图、算法伪代码、理论分析）

---

### Phase 3: 实验设计

**目标**：设计严谨、完整、可复现的实验方案

**核心原则**：
- 每个研究假设对应至少一组实验
- 统计显著性：3-5 次重复 + 均值/标准差
- 消融实验覆盖每个核心创新模块
- 结果展示遵循论文规范

**交付物**：
- `experiment-design.md` — 实验设计文档

---

### Phase 4: 框架搭建与步骤拆解

**目标**：选定基线框架，搭建项目结构，拆解实现步骤

**原则**：优先复用 > 二次修改 > 从零实现

**交付物**：
- `research-steps.json` — 原子化实现步骤

---

### Phase 5: 循环实现

**目标**：逐步执行所有实现步骤

**每步执行流程**：
1. 上下文研究 → 2. 任务规划 → 3. 代码实现 → 4. 自检 → 5. 测试 → 6. 摘要 → 7. 完成标记

---

### Phase 6: 实验规划与步骤拆解

**目标**：将实验设计文档中的所有实验拆解为原子化可执行步骤

**拆解顺序**：
1. 环境与数据准备
2. 基线方法实验（每个基线 × 数据集 = 一步）
3. 提出方法主实验
4. 消融实验（每个消融配置 = 一步）
5. 超参数敏感性搜索
6. 效率对比
7. 定性分析
8. 结果汇总表格
9. 图表生成
10. 报告撰写

**交付物**：
- `experiment-steps.json` — 原子化实验步骤

---

### Phase 7: 循环执行实验步骤

**目标**：逐步执行所有实验步骤，收集结果，生成图表和报告

**每步执行流程**：
1. 上下文研究 → 2. 任务规划 → 3. 执行实验/生成图表/撰写报告 → 4. 自检 → 5. 验证 → 6. 完成标记

**分阶段写入**：报告类步骤必须分段写入（每 ~200 行一次），防止超时。

## 注意事项

1. **不要跳过文献综述**：文献是创新的基础，综述质量直接影响后续方案质量
2. **优先复用现有框架**：不要从零开始造轮子，找到最接近的开源实现进行二次开发
3. **实验必须可复现**：所有随机种子、配置参数、环境版本必须记录
4. **结果要诚实**：如果方法在某些指标上不如基线，如实报告并分析原因
5. **BibTeX 必须完整**：每篇引用论文都要有标准的 BibTeX 条目
6. **UTF-8 编码**：所有文件必须使用 `edit`/`create_file` 工具写入，禁止终端重定向
7. **分阶段写入（防超时）**：所有长文档（综述、方法论、实验设计、报告、steps.json）必须分阶段多次调用工具写入，每次最多 200 行内容。先写入临时文件（`temp.md` 或 `temp.json`），每次调用确认上一段写入成功后再追加下一段，全部完成后重命名为最终文件名。两个 Loop 执行阶段（Phase 5、Phase 7）不需要此策略。

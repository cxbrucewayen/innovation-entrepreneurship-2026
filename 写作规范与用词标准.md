# 全国大学生创新创业大赛计划书写作规范与用词标准

## —— 面向金奖/一等奖水准的内容生成指南

> **使用说明**：凡 AI 生成计划书正文，均须严格遵循本文件规定的逻辑框架、用词风格与排版规范，以保证全书术语一致、论据充分、层次清晰，达到全国总决赛金奖水准。
>
> **⚠️ 强制约束——技术细节屏蔽规则**：参考材料（如 `创新创业计划书材料.md`）中的技术实现细节**仅供写作者理解产品能力，严禁原样搬入计划书正文**。具体禁止行为：写出具体步骤数（如"14步"）、暴露内部架构命名（模块数量/框架版本）、罗列底层技术栈型号。所有技术方案须转化为**用户价值语言**或**能力结果语言**（见第七节术语表）。评委关注的是"你解决了什么问题、效果有多好"，而非"你怎么实现的"。

## 一、整体写作哲学

### 1.1 核心原则（优先级从高到低）

| 优先级 | 原则                         | 说明                                                   |
| ------ | ---------------------------- | ------------------------------------------------------ |
| P1     | **数据驱动，证据为王** | 每一个判断必须附权威来源数据或可验证事实；禁止空泛形容 |
| P2     | **问题导向，解决闭环** | 先清晰定义痛点 → 再给出解决方案 → 再量化验证效果     |
| P3     | **价值主张明确唯一**   | 每章节有且仅有一个核心论点，段落间逻辑链条完整         |
| P4     | **差异化核心前置**     | 竞争壁垒、创新亮点须在章节开头或摘要位置点明           |
| P5     | **评委视角自审**       | 每段结束前自问："评委能否凭此段得出我期望的结论？"     |

### 1.2 禁止使用的表达

以下为评委高度反感的空泛表述，**一律禁止出现**：

- "前景广阔"、"市场巨大"（→ 必须替换为具体亿元级数据）
- "技术领先"、"国际先进"（→ 必须附专利号/测试报告/权威评测）
- "用户喜爱"、"好评如潮"（→ 必须附NPS分数/用户量/留存率数据）
- "团队优秀"（→ 必须列举获奖/发表/工作经历）
- "未来可期"、"潜力无限"（→ 必须替换为量化里程碑）
- "本项目旨在..."（→ 直接陈述事实或结论）

## 二、逻辑框架与章节结构

### 2.1 全书宏观叙事弧（黄金逻辑链）

```
【世界为什么需要我们】
→ 行业趋势驱动 + 用户痛点刚需
   ↓
【我们做了什么】
→ 产品核心功能 + 技术创新突破
   ↓
【我们比别人强在哪】
→ 竞品对比 + 不可复制壁垒
   ↓
【市场有多大，我们能拿多少】
→ TAM/SAM/SOM + 目标市场定位
   ↓
【我们怎么赚钱】
→ 商业模式 + 收入结构 + 盈利路径
   ↓
【我们凭什么做成】
→ 团队背景 + 资源禀赋 + 已验证成果
   ↓
【需要多少钱，怎么花，回报如何】
→ 融资计划 + 资金用途 + 财务预测
   ↓
【风险与应对】
→ 主要风险识别 + 应对预案
   ↓
【三年蓝图】
→ 阶段里程碑 + 社会价值
```

### 2.2 各章节核心任务与禁忌

#### 第一章：项目概述

- **核心任务**：90秒内让评委记住"是什么、有多厉害、凭什么信"
- **必须包含**：一句话项目定义 + 三大核心优势（数据佐证）+ 团队关键背景
- **禁忌**：项目背景写成"历史课"，技术描述过于底层，忘记回答"为什么是你们"

#### 第二章：市场分析

- **核心任务**：用权威数据构建"这是一个值得入局的千亿赛道"的共识
- **必须包含**：TAM/SAM/SOM三层漏斗 + 政策利好 + 增速数据 + 目标客群画像
- **禁忌**：数据无来源、市场边界不清晰、目标客群太泛

#### 第三章：产品与服务

- **核心任务**：让评委理解"你的产品做到了别人做不到的事"
- **必须包含**：核心功能列表 + 技术架构图 + 知识产权清单 + 用户使用路径图
- **禁忌**：功能罗列无重点，技术描述缺少"为什么难"的铺垫

#### 第四章：商业模式

- **核心任务**：证明有清晰可持续的盈利路径
- **必须包含**：价值创造→价值传递→价值捕获的完整闭环 + 单位经济模型（LTV/CAC）
- **禁忌**：收入模式不具体（如"广告+增值"无详细分解）

#### 第五章：营销与获客

- **核心任务**：展示低成本、高转化、可规模化的用户增长策略
- **必须包含**：增长飞轮图 + 渠道漏斗 + 已验证的流量数据
- **禁忌**：增长策略停留在"口号"层面（如"做好口碑营销"）

#### 第六章：创业团队

- **核心任务**：让评委相信"这支团队有能力把这件事做成"
- **必须包含**：核心成员背景（量化成就）+ 团队互补性分析 + 导师/顾问阵容
- **禁忌**：团队介绍千篇一律（"热爱创业、奋发向上"等空话）

#### 第七章：财务预测与融资计划

- **核心任务**：展示合理的增长逻辑与投资回报潜力
- **必须包含**：三年收支预测表 + Break-even时间点 + 本轮融资额/用途/估值逻辑
- **禁忌**：数据缺乏推演过程（直接写"第三年营收1亿"而不给出来源假设）

#### 第八章：风险与对策

- **核心任务**：主动展示风险意识，增强评委信任
- **必须包含**：3-5个主要风险（技术/市场/团队/政策）+ 具体应对预案
- **禁忌**：风险描述过于乐观自我安慰，或机械套用"风险矩阵"没有实质内容

#### 第九章：发展规划

- **核心任务**：描绘清晰可信的三年发展路线图
- **必须包含**：阶段性里程碑（时间+指标+关键动作）+ 社会价值阐述
- **禁忌**：目标脱离实际，时间节点模糊（如"近期""不久后"）

## 三、专业用词标准库

### 3.1 市场与行业分析用词

**描述市场规模时**（必须搭配数据）：

- `据艾瑞咨询/IDC/Frost & Sullivan/国家统计局数据显示`
- `市场规模已突破 X 亿元，预计 20XX 年达 X 亿元，CAGR（复合年均增长率）为 X%`
- `渗透率仍处于 X% 的低位，存在巨大的市场挖掘空间`
- `蓝海市场/细分赛道/垂直领域`
- `第一曲线增长/结构性机遇/赛道红利`

**描述政策环境时**：

- `政策东风/政策利好叠加`
- `纳入国家"十四五"规划/国务院X号文件明确提出`
- `监管合规路径清晰`
- `政策倾斜/产业扶持基金`

**描述用户痛点时**：

- `高频刚需痛点`、`决策链短`
- `现有解决方案存在明显天花板`
- `用户教育成本低`（指向刚需场景）
- `痛点量化：用户每 X 周期浪费 Y 小时/Y 元`

### 3.2 技术与产品用词

**描述技术壁垒时**（必须有选择地使用，不可滥用）：

- `自主研发/自主知识产权`
- `业界首创/率先落地`（必须附可验证依据）
- `核心专利保护/PCT国际专利申请中`
- `技术护城河/算法壁垒/数据飞轮效应`
- `突破 X 技术瓶颈，实现 X → Y 的性能跃迁`

**描述产品能力时**：

- `端到端自动化流水线`
- `全链路闭环`
- `毫秒级响应/亚秒级`
- `零代码/低门槛部署`
- `多模态融合`
- `SaaS化/PaaS化/平台化`
- `覆盖X类使用场景`

**描述系统能力时**（须转化为能力语言，禁止罗列架构/技术栈细节）：

- `多智能体协同处理，保障内容生成质量`
- `支持大规模并发，服务稳定可靠`
- `融合语义理解与知识图谱，实现精准内容定位`
- `弹性扩缩容，随用户增长平滑升级`

### 3.3 商业模式用词

**描述收入模式时**：

- `订阅制（Subscription）/ 积分消耗制（Credit-based）`
- `Freemium 免费增值模式`
- `平台佣金抽成`
- `B2B企业授权 / B2C直客`
- `API调用计费 / 按量付费`
- `复合变现模型`（指多条收入线并行）

**描述增长模式时**：

- `PMF（产品市场契合度）验证`
- `增长飞轮/正向循环`
- `病毒系数 K > 1`
- `分销裂变 / 私域流量运营`
- `用户LTV（终身价值）/ CAC（获客成本）`
- `NPS（净推荐值）`
- `DAU/MAU/GMV`

**描述竞争优势时**：

- `差异化竞争 / 错位竞争`
- `先发优势 / 网络效应`
- `规模壁垒 / 生态壁垒`
- `从"点状"到"体系化"的能力跃迁`

### 3.4 团队与执行用词

**描述团队背景时**：

- `具备X年行业深耕经验`
- `主导/参与过X项目，累计产生X亿元价值`
- `曾获/入选：XXX奖项/计划/基金`（具体名称）
- `技术型创始人 + 市场型联创的互补结构`
- `顾问阵容涵盖行业TOP X企业高管`

**描述执行进度时**（显示"我们不只是纸上谈兵"）：

- `已完成MVP（最小可行产品）开发并上线`
- `种子用户池已达X人，留存率X%`
- `完成首轮天使融资，估值X万元`
- `已与X家机构签订意向合作协议（MOU）`
- `软件著作权（软著）编号：XXXX`

### 3.5 财务与融资用词

**描述财务模型时**：

- `毛利率 / 净利率 / EBITDA`
- `盈亏平衡点（BEP）预计于X年X季度达成`
- `ROI（投资回报率）/ IRR（内部收益率）`
- `现金流正转时间`
- `人效比 / 坪效`

**描述融资需求时**：

- `本轮融资 X 万元，Pre-money 估值 X 万元`
- `资金用途：产品研发 X%，市场推广 X%，团队扩张 X%，运营储备 X%`
- `本融资可支撑 X 个月运营，实现 XX 里程碑`
- `参照同类项目 X 倍市销率/市盈率，估值合理`

## 四、句式与段落规范

### 4.1 标准论证句式

**陈述问题**（问题量化 → 根因分析）：

> "现有解决方案在 [场景] 中存在 [痛点]，导致用户每 [周期] 损失约 [X时间/X元]。根本原因在于 [底层逻辑缺失]，而非表面的 [表象问题]。"

**引出方案**（方案差异化 → 效果量化）：

> "[产品名] 通过 [核心机制/技术]，将 [指标A] 从 [原状态] 提升至 [目标状态]，效率提升 [X倍/X%]，已在 [验证场景] 中得到印证。"

**竞品对比**（维度明确 → 优势聚焦）：

> "与 [竞品] 相比，[产品名] 在 [关键维度] 上实现了 [差距量化]，核心差异来源于 [不可复制的能力点]，非竞品短期可追赶。"

**数据引用**（来源权威 → 结论明确）：

> "据 [权威来源/时间] 数据，[市场/用户指标] 为 [具体数值]，同比增长 [X%]，印证了 [核心判断]。"

### 4.2 段落写作标准

- **每段首句**：必须是该段的核心结论句（倒金字塔结构）
- **段落长度**：正文段落控制在80-150字；超过150字须拆分或改用列表
- **数据密度**：每200字正文中至少出现1个量化数据
- **图表配比**：每章节正文不超过500字处必须插入一张图/表辅助说明
- **加粗规则**：**每段至多加粗1-2个关键词**，全章加粗词汇不超过10个（防止视觉噪音）

### 4.3 标题层级规范

```
# 第X章 章节名称（二号黑体）
## X.X 一级小节（三号黑体）
### X.X.X 二级小节（小三号黑体）
#### X.X.X.X 三级条目（四号黑体，正文可不独立成节）
- 列表项（正文五号宋体/雅黑）
```

### 5.2 表格写作规范

- 表头必须加粗
- 对比表中"本项目"一列须加粗或特殊标注（**LumaMemo** 或 ★）
- 数据单元格右对齐，文字内容左对齐
- 表格下方必须注明"数据来源：XX，统计截至 XXXX 年"

### 5.3 ASCII 图示规范

在 Markdown 中使用代码块（\`\`\`）绘制框图，要求：

- 对齐工整，使用 `─ │ ┌ ┐ └ ┘ ┬ ┴ ├ ┤ ┼`等 Unicode 框线字符
- 箭头使用 `→` `↓` `↑` `←`（不使用 `->`）
- 每个框内文字居中

## 六、特定章节深度规范

### 6.1 竞品分析（高评分要点）

评委看重的不是"我们比竞品好"的结论，而是**分析框架的专业性**：

1. **维度选取**：从用户最核心需求出发选取6-8个维度（非功能堆砌）
2. **竞品选择**：包含直接竞品（同类型）+ 间接竞品（替代方案）+ 潜在巨头威胁
3. **数据标注**：竞品数据须有出处（官网/第三方测评/媒体报道）
4. **护城河分析**：最后1-2段须明确指出"竞品在哪个维度短期无法追赶，原因是X"

### 6.2 财务预测（高评分要点）

1. **假设前置**：在预测表前列出所有关键假设（用户增长率/ARPU/毛利率/市场费用率）
2. **情景分析**：给出保守/基准/乐观三套预测
3. **资金用途细化**：细化到产品/市场/人力/行政四大类，各占比精确到个位数
4. **估值逻辑**：明确采用哪种估值方法（市销率/DCF/可比公司法）并给出计算过程

### 6.3 项目概述（高评分要点）

1. **"电梯演讲"密度**：100字内完成"平台是什么 → 解决了什么问题 → 核心亮点是什么 → 目前进展如何"
2. **关键数字前置**：至少3个亮眼且可信的数字（用户量/技术指标/市场规模/融资额）
3. **创新维度**：区分技术创新/模式创新/应用创新，不可混淆

## 八、写作流程 SOP（供 AI 生成内容时遵循）

```
Step 1: 明确本段/本节的【核心论点】（一句话）
           ↓
Step 2: 收集支撑该论点的【3个以上数据或事实】
           ↓
Step 3: 用【标准论证句式】组织段落，首句放结论
           ↓
Step 4: 检查是否存在【禁止用词】，替换为专业表述
           ↓
Step 5: 确认与【术语统一表】保持一致
           ↓
Step 6: 补充对应的【图表/数据表】（如该段有图表需求）
           ↓
Step 7: 自检评委视角：能否凭此段给出满分判断？
```

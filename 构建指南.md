# 创新创业计划书 LaTeX 构建指南

## 编译环境

- **TeX 发行版**：TeX Live 2025（推荐）或 MiKTeX
- **编译引擎**：XeLaTeX（必须，因为用了中文字体和 `.otf`/`.ttf` 字体文件）
- **工作目录**：`innovation-entrepreneurship-2026/`
- **主文件**：`main.tex`
- **模板类文件**：`cxcyplan.cls`

## 环境安装（本机未安装 LaTeX）

### 方案一：安装 TeX Live 2025（推荐，约 7GB 完整版）

1. 前往 [https://www.tug.org/texlive/](https://www.tug.org/texlive/) 下载 `install-tl-windows.exe`
2. 运行安装程序，建议选择"完整安装"（避免缺包问题）
3. 安装完成后在 PowerShell 中验证：
   ```powershell
   xelatex --version
   ```

### 方案二：安装 MiKTeX（约 300MB 基础包，按需下载）

1. 前往 [https://miktex.org/download](https://miktex.org/download) 下载安装
2. 首次编译时会自动提示下载缺失宏包，需联网

### 方案三：VS Code + LaTeX Workshop 扩展（强烈推荐，配合命令行使用）

1. 在 VS Code 扩展市场搜索 `LaTeX Workshop`（作者 James Yu）并安装
2. 安装完 TeX Live 后，打开 `main.tex`，点击右上角 ▶ 即可一键编译
3. 编译配方选择 `xelatex`（LaTeX Workshop 默认支持）

## 先确认你在用哪个终端

- 本文中的 `Get-ChildItem`、`Remove-Item`、`Write-Host` 是 **PowerShell** 命令。
- 如果你在 **cmd** 里直接执行 PowerShell 命令，会报错或不生效。
- 建议在 VS Code 里切到 PowerShell 终端（终端 → 新终端，默认即 PowerShell）。

## 完整编译流程（3步）

本项目**默认不启用参考文献**（`main.tex` 中 `\iffalse...\fi` 包裹了 bibtex 相关命令），因此只需 3 步：

在 `innovation-entrepreneurship-2026/` 目录下执行：

```powershell
# 切换到项目目录
Push-Location "d:\vscode\lumamemo\国际大学生创新创业2026\innovation-entrepreneurship-2026"

# 1. 清理中间文件（首次编译或出错时执行）
Remove-Item -Force *.aux, *.log, *.toc, *.lof, *.lot, *.out, *.fls, *.fdb_latexmk, *.synctex.gz, main.pdf -ErrorAction SilentlyContinue

# 2. 第一次 xelatex（生成 aux / toc）
xelatex -interaction=nonstopmode main.tex

# 3. 第二次 xelatex（解析目录、交叉引用、页码）
xelatex -interaction=nonstopmode main.tex

Pop-Location
```

### 一键编译脚本（复制粘贴即用）

```powershell
Push-Location "d:\vscode\lumamemo\国际大学生创新创业2026\innovation-entrepreneurship-2026"
Remove-Item -Force *.aux, *.log, *.toc, *.lof, *.lot, *.out, *.fls, *.fdb_latexmk, *.synctex.gz, main.pdf -ErrorAction SilentlyContinue
xelatex -interaction=nonstopmode main.tex | Out-Null
xelatex -interaction=nonstopmode main.tex | Out-Null
Write-Host "=== 编译完成 ===" -ForegroundColor Green
if (Test-Path main.pdf) { Write-Host ("PDF 大小：{0:F2} MB" -f ((Get-Item main.pdf).Length / 1MB)) }
Pop-Location
```

### cmd 版本（如果只有 cmd 环境）

在 `innovation-entrepreneurship-2026\` 目录下执行：

```cmd
del /q *.aux *.log *.toc *.lof *.lot *.out *.fls *.fdb_latexmk *.synctex.gz main.pdf

xelatex -interaction=nonstopmode main.tex
xelatex -interaction=nonstopmode main.tex
```

## 启用参考文献（可选）

默认情况下参考文献编译被 `\iffalse...\fi` 禁用。如需启用，打开 `main.tex`，将以下部分的 `\iffalse` 和 `\fi` 删除或注释：

```latex
% 启用前（当前默认状态）：
\iffalse
\backmatter
...
\makereferences
\fi

% 启用后（删掉 \iffalse 和 \fi）：
\backmatter
...
\makereferences
```

启用后需执行完整 4 步编译：

```powershell
xelatex -interaction=nonstopmode main.tex | Out-Null
bibtex main
xelatex -interaction=nonstopmode main.tex | Out-Null
xelatex -interaction=nonstopmode main.tex | Out-Null
```

## 增量编译（日常最常用）

**不需要每次都清理+完整编译！** 根据改动内容选择策略：

| 改了什么 | 需要执行的命令 |
| -------- | -------------- |
| 只改了正文文字/公式/表格 | `xelatex main.tex`（1 次即可） |
| 改了交叉引用（`\ref`）或目录 | `xelatex` × 2 |
| 出现编译错误、格式异常 | 完整清理 + 重新编译 2 次 |

```powershell
# 日常最常用：只改了文字，一次即可
xelatex -interaction=nonstopmode main.tex | Out-Null
```

## 常见问题

### 编译后交叉引用显示 `??`
需要再运行一次 `xelatex`。第一次编译只生成 `.aux` 记录引用，第二次才能正确解析。

### 中文字体找不到 / `fontspec` 报错
确保系统已安装 Times New Roman（Windows 系统通常自带）。项目中文字体走 `fonts/` 目录下的 `.otf` 文件，无需额外安装。若仍报错，检查 `fonts/` 目录是否完整。

### `xelatex` 命令找不到
TeX Live 安装后需重启终端（或重新打开 VS Code），让 PATH 环境变量生效。也可以手动将 TeX Live 的 `bin/windows/` 目录加入系统 PATH。

### 编译输出中有大量警告但能生成 PDF
`nonstopmode` 下警告不影响输出，可忽略。如果 PDF 样式异常，再检查具体警告内容。

### 想只预览某一章

`main.tex` 使用 `\input` 加载各章，可临时注释掉不需要的章节加速预览：

```latex
\input{chap01}
% \input{chap02}   ← 临时注释掉不编译的章
\input{chap03}
```

> 预览完记得恢复注释，提交前确保所有章节都已启用。

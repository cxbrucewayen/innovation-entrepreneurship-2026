# WZU 论文 LaTeX 构建指南

## 编译环境

- **TeX 发行版**：TeX Live 2025
- **编译引擎**：XeLaTeX
- **工作目录**：`WZUthesis20260326/`
- **主文件**：`main.tex`

## 先确认你在用哪个终端

- 本文中的 `Get-ChildItem`、`Remove-Item`、`Write-Host` 是 **PowerShell** 命令。
- 如果你在 **cmd** 里直接执行 PowerShell 命令，会导致"看起来执行了，但实际没清干净"。
- 建议：
	- 要么在 VS Code 里切到 PowerShell 再执行；
	- 要么用下方提供的 **cmd 版本**命令。

## 完整编译流程（4步）

在 `WZUthesis20260326/` 目录下依次执行：

```powershell
# 1. 清理所有中间文件（关键！避免残留损坏的 aux 文件）
Get-ChildItem -Recurse -Include *.aux | Remove-Item -Force
Remove-Item -Force *.log, *.toc, *.lof, *.lot, *.out, *.bbl, *.blg, *.fls, *.fdb_latexmk, *.bcf, *.run.xml, *.synctex.gz, main.pdf -ErrorAction SilentlyContinue

# 2. 第一次 xelatex（生成 aux 文件，含 \bibdata 和 \citation）
xelatex -interaction=nonstopmode main.tex | Out-Null

# 3. bibtex（处理参考文献）
bibtex main

# 4. 再跑两次 xelatex（解析交叉引用、目录、页码）
xelatex -interaction=nonstopmode main.tex | Out-Null
xelatex -interaction=nonstopmode main.tex | Out-Null
```

## 草稿模式（加速日常编写）

在 `main.tex` 中有以下行控制草稿模式，启用后图片显示为占位框，编译速度大幅提升：

```latex
% 启用草稿模式（日常编写时使用）
\setkeys{Gin}{draft=true}

% 关闭草稿模式（最终提交时注释掉上面这行）
% \setkeys{Gin}{draft=true}
```

**注意**：草稿模式控制在 `main.tex` 中统一管理，不要在章节文件里局部切换。

## 盲审模式

`main.tex` 中有以下控制行：

```latex
%\def\blind{}       % 盲审版把这条注释打开
```

- **正式版**（显示正常封面和页眉）：保持注释状态（`%\def\blind{}`）
- **盲审版**（隐去姓名、导师等信息）：取消注释（`\def\blind{}`）

## 加速编译技巧

> 完整 4 步编译需要跑 3 次 `xelatex`，每次都要读取和嵌入全部图片，是编译慢的主要原因。

### 技巧 1：日常编写用草稿模式（最有效，提速 10 倍+）

在 `main.tex` 中确保以下行未被注释：

```latex
\setkeys{Gin}{draft=true}
```

> 只修改文字/公式/表格时，始终用草稿模式，只在需要检查图片排版或最终提交时关闭。

### 技巧 2：增量编译（日常最常用）

**不需要每次都清理+完整 4 步编译！** 根据你改了什么选择编译策略：

| 改了什么              | 需要执行的命令                        |
| --------------------- | ------------------------------------- |
| 只改了正文文字/公式   | `xelatex main.tex`（1 次即可）        |
| 改了交叉引用/目录     | `xelatex` × 2                         |
| 改了参考文献 main.bib | `xelatex` → `bibtex main` → `xelatex` × 2 |
| 编译报错/格式异常     | 完整清理 + 4 步编译                   |

```cmd
@REM 日常最常用：只改了文字，跑一次就够
xelatex -interaction=nonstopmode main.tex > nul
```

### 技巧 3：使用 `\include` + `\includeonly` 只编译当前章节

当前论文用 `\input` 加载章节，无法跳过未修改的章。改为 `\include` + `\includeonly` 可以实现 **只编译你正在编辑的那一章**，其余章节保持上次编译结果。

**第一步：将 `main.tex` 中正文章节的 `\input` 改为 `\include`**

```latex
% 正文部分 — 改为 \include（注意去掉 .tex 后缀）
\include{chapters/chap01}
\include{chapters/chap02}
\include{chapters/chap03}
\include{chapters/chap04}
\include{chapters/chap05}
\include{chapters/chap06}
```

**第二步：在 `\begin{document}` 前加 `\includeonly`，只编译你要的章节**

```latex
% 只编译第3章（其余章节保持上次结果，页码不变）
\includeonly{chapters/chap03}

\begin{document}
```

> **注意**：`\includeonly` 只对 `\include` 生效，对 `\input` 无效。
> `\include` 会在每章前后插入 `\clearpage`（新起一页），对论文排版通常没有影响。
> 需要完整编译时，注释掉 `\includeonly` 那一行即可。

### 技巧 4：`-no-pdf` 模式加速中间编译

在多次 `xelatex` 编译中，前面几次不需要真正生成 PDF，只需要处理交叉引用：

```cmd
@REM 第1次和第2次不生成 PDF（只输出 xdv）
xelatex -no-pdf -interaction=nonstopmode main.tex > nul
bibtex main
xelatex -no-pdf -interaction=nonstopmode main.tex > nul
@REM 最后一次才生成 PDF
xelatex -interaction=nonstopmode main.tex > nul
```

## 常见问题

### 编译后交叉引用显示 `??`
需要再运行一次 `xelatex`，或者执行完整 4 步流程。

### 参考文献显示 `[?]` 或 `[Und]`
未运行 `bibtex`，执行完整 4 步流程即可。

### `\backmatter` 报 `Use of \@ doesn't match` 错误
检查 `wzuthesis.cls` 中是否在 `.cls` 文件内部出现了 `\makeatother`（在 `.cls` 文件中 `@` 默认是字母，不需要 `\makeatletter`/`\makeatother` 包装）。

### 封面显示"XXX硕士学位论文"而非正式封面
检查 `main.tex` 第59行左右，确保 `\def\blind{}` 已注释：
```latex
%\def\blind{}       % 盲审版把这条注释打开
```
